## 2. 阶段A：简历上传、解析、结构化、可编辑、持久缓存
### 2.1 输入
- 用户在插件内**手动上传**本地简历文件  
- 支持：DOCX、TXT、XLSX

### 2.2 AI解析规则（强制）
1. **不限定字段**：无论简历包含什么信息，LLM 都要**自动识别、自动整理**。  
2. **多条目严格拆分**：  
   - 实习经历、项目经历、证书、奖项、校园经历等  
   - 每条独立成项  
   - 同一条内信息**不混杂、不串项、不与其他条目混淆**  
3. 输出**结构化嵌套JSON**，支持数组类型多条目。

### 2.3 缓存与版本
1. 解析后数据**持久缓存**，直到用户**主动删除**。  
2. 用户可在插件：  
   - 查看结构化信息  
   - 修改任意字段  
   - 新增/删除条目（实习、项目等）  
   - 编辑后**自动覆盖缓存**  
3. 插件支持**选择历史版本**进行填表。

### 2.4 输出
`local_resume_data.json`

## 3. 阶段B：网页表单智能识别
### 3.1 输入
- 当前网页所有可见表单元素

### 3.2 识别范围
- input（文本、电话、邮箱、日期等）  
- select 下拉框  
- textarea 多行文本  
- radio / checkbox 单选/复选  
- 层级联选下拉框（省市区、行业→职位等）

### 3.3 跳过规则
- 跳过**密码框**  
- 跳过**隐藏输入框**  
- 折叠/未展开的表单项：**暂不处理，用户展开后再填写**

### 3.4 输出
`web_form_structure.json`  
包含：标签文字、控件类型、name/id、是否已有内容、是否可见

## 4. 阶段C：字段语义映射 + 智能自动填写
### 4.1 输入
- `local_resume_data.json`  
- `web_form_structure.json`

### 4.2 字段映射
- 由 **LLM 纯语义匹配**，不写死规则  
- 不需要向量模型、不需要向量库

### 4.3 填写核心规则（最高优先级）
1. 用户在**点击开始前已手动填写**的表单项：**绝对不修改、不覆盖**。  
2. 网页原生 placeholder **不算已填写**。  
3. 插件之前填写、用户**未修改**的内容：**不再重复填写**。  
4. 识别失败/无对应信息：**留空**。

### 4.4 下拉与级联下拉
1. 普通下拉：自动匹配并选择。  
2. 级联下拉（省市区、行业→职位等）：  
   - 能直接选最后一级 → 直接选  
   - 必须按层级 → 按顺序选择  
   - **自动等待异步加载**  
   - 自动检查是否选中成功  
   - 需要点击确认 → 自动确认

### 4.5 暂停 / 继续
1. 支持：开始 → 暂停 → 继续  
2. 继续时**从断点继续**，不从头执行  
3. 同一网站内刷新：**保留暂停状态**  
4. 跳转到不同页面：**重置状态**

### 4.6 界面显示
- 小字显示**填写进度**：已填 X / 总数 Y  
- 日志面板：可展开、可收起、可调节高度
