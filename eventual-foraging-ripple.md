# 简历自动填写插件 - 开发需求文档

## 一、项目概述

### 1.1 项目名称
本地简历智能解析 → 网页表单自动填写插件（浏览器扩展 + 本地Python后端）

### 1.2 核心目标
1. 用户上传本地简历 → AI 自适应结构化解析
2. 识别任意招聘网页表单字段
3. 智能字段语义映射 + 安全自动填写
4. 支持下拉框、单选复选、暂停继续、不覆盖已有内容

### 1.3 目标浏览器
跨浏览器兼容（Chrome、Edge、Firefox等主流浏览器）

### 1.4 整体架构
```
┌───────────────────────┐         ┌─────────────────────────┐
│     浏览器插件界面      │ ←→      │  本地Python后端(GitHub)  │
│  • 上传/编辑/版本选择   │         │  • 文件解析              │
│  • 开始/暂停/继续       │         │  • LLM结构化抽取         │
│  • 悬浮/置顶/可拖拽     │         │  • 表单识别              │
└───────────┬─────────────┘         │  • 字段语义映射          │
            │                       └────────────┬────────────┘
            │                                    │
            ▼                                    ▼
┌─────────────────────────┐         ┌─────────────────────────┐
│     招聘/企业官网表单     │         │        LLM API          │
└─────────────────────────┘         │   (支持多家API提供商)     │
                                    └─────────────────────────┘
```

---

## 二、全局通用规则

1. **三阶段独立开发**：阶段A、B、C完全独立开发、独立测试、最后合并
2. **数据格式**：所有数据交换使用标准JSON，无多余文本
3. **不编造信息**：AI绝对不编造信息，识别不出则留空
4. **不覆盖原则**：自动填写只填空项，不覆盖、不修改已有内容
5. **悬浮窗特性**：可拖动、可调整大小、可置顶、点击外部不消失
6. **本地运行**：Python后端代码托管GitHub，在用户本地运行，不上传用户数据
7. **API Key配置**：LLM API Key配置在系统环境变量，不硬编码、不通过插件输入
8. **支持格式**：DOCX / TXT / XLSX

---

## 三、阶段A：简历上传、解析、结构化、可编辑、持久缓存

### 3.1 输入
- 用户在插件内手动上传本地简历文件
- 支持格式：DOCX、TXT、XLSX

### 3.2 AI解析规则
1. **不限定字段**：无论简历包含什么信息，LLM自动识别、自动整理
2. **多条目严格拆分**：
   - 实习经历、项目经历、证书、奖项、校园经历等
   - 每条独立成项
   - 同一条内信息不混杂、不串项、不与其他条目混淆
3. **输出格式**：结构化嵌套JSON，支持数组类型多条目

### 3.3 缓存与版本
1. 解析后数据持久缓存，直到用户主动删除
2. 用户可在插件中：
   - 查看结构化信息
   - 修改任意字段
   - 新增/删除条目（实习、项目等）
   - 编辑后自动覆盖缓存
3. 支持选择历史版本进行填表

### 3.4 输出
`local_resume_data.json`

---

## 四、阶段B：网页表单智能识别

### 4.1 输入
- 当前网页所有可见表单元素

### 4.2 识别范围
- input（文本、电话、邮箱、日期等）
- select 下拉框
- textarea 多行文本
- radio / checkbox 单选/复选

### 4.3 跳过规则
- 跳过密码框
- 跳过隐藏输入框
- 折叠/未展开的表单项：暂不处理，用户展开后再填写

### 4.4 输出
`web_form_structure.json`
包含：标签文字、控件类型、name/id、是否已有内容、是否可见

---

## 五、阶段C：字段语义映射 + 智能自动填写

### 5.1 输入
- `local_resume_data.json`
- `web_form_structure.json`

### 5.2 字段映射
- 由LLM纯语义匹配，不写死规则
- 不需要向量模型、不需要向量库

### 5.3 填写核心规则（最高优先级）
1. 用户在点击开始前已手动填写的表单项：绝对不修改、不覆盖
2. 网页原生placeholder不算已填写
3. 插件之前填写、用户未修改的内容：不再重复填写
4. 识别失败/无对应信息：留空

### 5.4 下拉框处理（V1版本）
- 普通下拉：自动匹配并选择
- 级联下拉（省市区、行业→职位等）：**V1暂不支持，后续迭代**

### 5.5 暂停 / 继续
1. 支持：开始 → 暂停 → 继续
2. 继续时从断点继续，不从头执行
3. 同一网站内刷新：保留暂停状态
4. 跳转到不同页面：重置状态

### 5.6 界面显示
- 小字显示填写进度：已填 X / 总数 Y
- 日志面板：可展开、可收起、可调节高度

---

## 六、浏览器插件界面要求

1. 小悬浮窗口，置顶显示，不被网页覆盖
2. 可拖动、可调整大小、点击外部区域不关闭、不隐藏
3. 功能按钮：
   - 上传简历
   - 编辑信息
   - 选择历史版本
   - 开始 / 暂停 / 继续
4. 实时日志区（可折叠）
5. 填写进度小字展示

---

## 七、Python 后端与 GitHub 部署规则

1. 后端完整代码托管在GitHub仓库
2. 用户自行下载/克隆到本地运行
3. 启动方式：`python server.py`，开启localhost本地服务
4. 不提供自动安装脚本，提示用户手动安装依赖
5. 数据流向：`插件 ↔ 本地后端 ↔ LLM API`
6. 不上传任何数据到GitHub

### 7.1 LLM API 支持
- 支持多家API提供商
- API Key 通过环境变量配置
- 用户可自行选择和切换

---

## 八、AI 行为强约束

1. 不编造、不联想、不扩充、不创作
2. 多条目必须结构化、独立、不混杂
3. 输出严格JSON，无多余解释、无markdown、无对话
4. 无法识别 → 留空
5. 只做语义理解、抽取、匹配

---

## 九、核心原则

> **无论用户输入了哪些字段，AI都可以识别并整理好，对多条的字段也能够进行区分，在后续填写表单时，已经填入且用户没有修改的就不再重新填入。**

---

## 十、交付物

1. 浏览器插件（前端界面 + 交互）

3. GitHub仓库结构与运行说明
4. 三阶段独立可运行、可测试、可合并

- 表单元素识别

- 语义映射逻辑
- 自动填写执行
- 暂停/继续机制
